### Phase 3：记录踩坑经验（持续）

**目标**：将遇到的问题整理成可搜索的知识库

**格式要求**：
```markdown
# [问题标题]

## 现象
- 发生了什么错误？
- 错误信息是什么？
- 在什么场景下出现？

## 根因分析
- 为什么会发生？
- 是约束不足？规则冲突？还是智能体误解？

## 解决方案
- 临时修复：...
- 长期方案：...

## 预防措施
- 如何避免再次发生？
- 需要添加什么约束或检查？

## 参考
- 相关 PR/Issue 链接
```

## ⚙️ 执行规则

1. **先读后写** — 修改任何文件前，先阅读相关上下文
2. **小步快跑** — 每次只改一个文件，提交前自测
3. **机械化验证** — 所有修改必须通过 lint/test/build
4. **记录反馈** — 遇到问题立即记录到 `feedback/`
5. **主动汇报** — 每完成一个概念文档，主动汇报进度
