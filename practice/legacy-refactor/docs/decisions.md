# 重构决策记录

## 2024-01-15: 技术栈选择

### 背景
旧代码库同时使用 Python Flask 和 Node.js Express，导致维护困难。

### 决策
- **后端**: Python FastAPI
  - 理由：性能更好，自动 API 文档，类型提示
- **前端**: TypeScript + Next.js
  - 理由：类型安全，SEO 友好，生态系统成熟

### 权衡
- ❌ 需要重新学习 FastAPI
- ✅ 长期维护成本降低
- ✅ 开发效率提升

## 2024-01-16: 数据库迁移策略

1. 创建 SQLAlchemy 模型
2. 编写 Alembic 迁移脚本
3. 测试数据完整性
4. 逐步切换流量

## 踩坑记录

### 坑 1: 异步函数混用

**问题**: FastAPI 中混用同步和异步函数导致性能问题。

**解决**: 
```python
# ❌ 错误
def get_user(id: int):
    return await db.query(...)

# ✅ 正确
async def get_user(id: int):
    return await db.query(...)
```

### 坑 2: 循环依赖

**问题**: 模块间循环依赖导致导入错误。

**解决**: 
- 提取共享代码到单独模块
- 使用依赖注入
- 重构为清晰的分层架构

### 坑 3: 测试数据污染

**问题**: 测试间共享数据库导致结果不稳定。

**解决**: 
```python
@pytest.fixture
def db_session():
    session = create_test_session()
    yield session
    session.rollback()  # 清理
```
